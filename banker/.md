C. The Budget Service (The State)
Role: Tracks spending so a campaign doesn't overspend.

Go Flex: Use Atomic operations (sync/atomic) or Lua scripts in Redis to handle "checks and balances" without race conditions.

The Banker (The Ledger)
Accepts: GET /balance/{campaign_id} and POST /spend.

Does:

Uses Atomic Counters (in-memory) or Redis INCRBY to ensure budget isn't double-spent.

High-level move: Implement a "Local Cache" in the Bidder that syncs with the Banker every 100ms to reduce network calls (Budget Pacing).

Returns: Boolean (Is allowed?) or the remaining balance.

GET,/health,K8s Liveness Check,Plain Text
POST,/accounts,Create new campaign wallet,JSON
PUT,/accounts/{id}/balance,Add/Update funds (Top-up),"JSON { ""amount"": 5000000 }"
